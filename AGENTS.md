# Repository Guidelines

## Project Structure & Module Organization

PartSync is a Vite web app with a Node.js SQLite API.

- `index.html` defines the app shell and loads the frontend bundle.
- `src/main.js` contains UI state, routing, forms, history actions, and CSV export.
- `src/api.js` centralizes calls to `/api/*` endpoints.
- `src/style.css` contains the design system, layout, and responsive styles.
- `server.js` starts the HTTP server, Vite middleware, production static serving, and API routes.
- `server/database.js` owns SQLite setup, schema creation, validation, and persistence.
- `data/` stores local SQLite runtime files and is ignored by Git.
- `dist/` is generated by Vite and should not be edited directly.

## Build, Test, and Development Commands

- `npm install` installs the project dependencies.
- `npm run dev` starts the local app with Vite middleware and SQLite API, usually at `http://localhost:5173`.
- `npm run build` creates the production frontend bundle in `dist/`.
- `npm run start` serves the built app and API in production mode.
- `npm run preview` builds and then starts the production server.

The server uses Node's experimental SQLite support, so use a Node.js version that provides `node:sqlite`.

## Coding Style & Naming Conventions

Use ES modules, two-space indentation, semicolons, and double quotes for JavaScript. Prefer descriptive camelCase identifiers such as `saveRequests`, `deviceModel`, and `createdAt`. Keep DOM selectors and element IDs aligned with `index.html`; when adding UI controls, wire them from `src/main.js` near related setup functions. CSS uses custom properties in `:root`; add new colors or spacing tokens there before scattering literals.

## Testing Guidelines

There is no automated test suite configured yet. Before submitting changes, run `npm run build` and manually verify the main flow with `npm run dev`: create a request, add multiple parts, update status, filter history, export CSV, save settings, and confirm persistence after restart. For API changes, check `/api/health` and the affected endpoint.

## Commit & Pull Request Guidelines

Recent commits use short, sentence-style summaries, often in Portuguese, for example `Adiciona servidor Node e API com suporte a SQLite`. Keep commits focused and describe the visible change. Pull requests should include a concise summary, manual verification steps, linked issue if applicable, and screenshots or short clips for UI changes.

## Security & Configuration Tips

Do not commit local databases, `node_modules/`, `dist/`, or `.env*` files. Use `PARTSYNC_DATA_DIR` or `PARTSYNC_DB_PATH` to point the SQLite database at a custom local location when needed.
